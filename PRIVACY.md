# Privacy Policy (Draft)

Last updated: 2026-03-13

## Data Processed

The app can process:

- API credentials configured by the user
- Trading telemetry (positions, orders, P&L, logs)
- Local UI preferences (accent color, chat history)

## Storage

- Backend config is stored locally in `kalshi-config.json` and/or environment variables.
- Frontend preferences are stored in browser local storage under `kalshi-ui-store`.
- Runtime logs are stored locally in `kalshi-agent.log`.

## Data Sharing

- The app does not intentionally transmit user data to third parties except configured market/data APIs and model providers required for operation.

## Security

- Dashboard defaults to localhost binding.
- Optional dashboard token can be required for control endpoints.
- Users are responsible for securing host access and credential files.

## User Responsibility

- Users are responsible for legal, tax, regulatory, and trading compliance in their jurisdiction.
- Users are responsible for validating dry-run behavior before enabling live mode.
